### 💡 选型参考与重要说明

1.  **整数类型选择**：`int` 是最常用的整数类型，除非有特殊的内存限制或数值范围要求，否则通常建议使用 `int`。当 `int` 的范围不足时，再考虑使用 `long` 。
2.  **浮点数精度权衡**：`float` 是单精度浮点数，`double` 是双精度浮点数。`double` 具有更高的精度，是大多数科学计算和通用浮点运算的首选。`decimal` 则提供了更高的精度和更小的范围，适用于财务、货币等不允许舍入误差的精确计算 。
3.  **有符号与无符号**：名称以 `u` 开头的类型（如 `uint`、`ulong`）是**无符号整数**，只能表示非负数，但其正数表示范围比对应的有符号类型（如 `int`、`long`）更大 。
4.  **字面量后缀**：为数值字面量指定类型时，可以使用后缀，例如 `12.3F` 表示 `float`，`12.3D` 表示 `double`，`12.3M` 表示 `decimal`，`100L` 表示 `long` 。如果直接写 `12.3`，编译器会默认其为 `double` 类型 。
5.  **类型转换**：C#支持隐式转换（如将 `int` 赋值给 `long`）和显式转换（如使用 `(int)` 进行强制转换）。此外，还可以使用 `Convert.ToInt32()`、`int.Parse()` 和 `int.TryParse()` 等方法进行转换，它们在处理 `null` 值和转换失败时的行为有所不同 。

## 转义字符

### 补充说明与技巧

*   **`@`符号的妙用（逐字字符串）**：在字符串字面量前加上 `@` 前缀，可以创建“逐字字符串”。在这种字符串中，转义字符（除了 `"` 之外）不会被处理，反斜杠会被当作普通字符。这在处理文件路径或正则表达式时特别方便。
    ```csharp
    // 使用普通字符串，需要转义反斜杠
    string path1 = "C:\\Users\\Name\\Documents";
    // 使用逐字字符串，更加清晰
    string path2 = @"C:\Users\Name\Documents";
    
    // 逐字字符串还可以直接跨越多行
    string multiLineText = @"第一行
    第二行
    第三行";
    
    // 在逐字字符串中表示双引号，需要用两个双引号""
    string quote = @"""Hello, World!"" he said.";
    ```

*   **`$`与`@`的组合使用**：可以将 `$`（用于字符串插值）和 `@` 结合使用，顺序可以是 `$@"..."` 或 `@$"..."`。这样可以在保留逐字字符串特性的同时，嵌入变量或表达式。
    ```csharp
    string name = "Alice";
    string message = $@"欢迎访问 {name}!
    您的文件路径是：C:\Users\{name}\file.txt";
    ```+-

## 类型转换

学习完类型后，对类型的相互转换就是工作中常见的需求，类型转换分为隐式转换（不需要你写转换方法，编译器自动完成的转换），显示转换（开发人员明确写出转换的方法）

### 隐式转换

先说隐式转换的原则是 范围大的包括范围小的，即范围小的可以隐式转换成为范围的
#### 有符号整形之间转换
long>int>short>sbyte
#### 无符号整数
ulong>uint>ushort>byte 
注意 有符号与无符号之间，范围包含就可以转换，但是范围小的不可以转换成范围大的
比如 int 不能转换成 uint，因为int 的范围不能包含uint ，因为有负数
整型数字可以转换成浮点数，因为浮点数的范围很大，可以包含整数的范围
能不能换换核心要看范围是否包含
float和double 都可以存储 整形

### 显示转换
#### 括号显示转换
显示转换即强制转换，使用（type）类型符号来将强制转换
核心要注意 范围问题，和精度问题，要非常小心，因为转换不报错但是结果不一定是你想要的，
可能是无意义的
整数强转 范围和精度，范围异常和精度丢失（不会四舍五入）
#### Parse法
用于将字符串转换成对应类型的方法
当然程序要知道可以转换，如何不能转换的情况下会报错异常
int.Parse("123") 将123 转换成 int 123，int.Parse("fdafd") 会报错
#### Convert 法
使用系统提供的 Convert.ToInt32("123") 将字符串转换成 int 123
实际上是系统提供的通用的转换方法，也是要求数据本身可以转换，否则会异常
区别 Convert 会四舍五入
